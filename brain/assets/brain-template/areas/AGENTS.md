# Areas: Agent guidance

Guidance for agents working under **areas/** (folder-level). Repo-wide rules apply: see [AGENTS.md](../AGENTS.md) for conventions and layout.

## Scope

- **This scope covers:** The areas directory and all area folders. Areas are ongoing responsibilities with no fixed "done" date (work, learning, relationships, etc.).
- **Per-area rules:** Each area folder (e.g. `template-area/`) should have its own README.md and AGENTS.md. When working inside a specific area, also load that area’s AGENTS.md (it overlays this file and root).

## Conventions

- One folder per area; name folders in kebab-case.
- When creating a new area, copy from [template-area/](template-area/) and adapt README.md and AGENTS.md.
- Do not put time-bound or outcome-bound work here; that belongs in [projects/](../projects/). Do not put atomic concept notes here; those belong in [zettelkasten/](../zettelkasten/).

## Procedures

Each procedure has a **name**, then **When** and **What to do** as section headers. **What to do** is an enumerated list (1. 2. 3. …).

### Create a new area

#### When

The user or context requires a new area (e.g. a new ongoing responsibility).

#### What to do

1. Copy [template-area/](template-area/) to a new folder under areas/ with a kebab-case name.
2. Adapt the new folder’s README.md (what this area is, how to use it).
3. Adapt the new folder’s AGENTS.md (scope, conventions, procedures with name, When and What to do as section headers, What to do as an enumerated list; how to operate).
4. Do not add time-bound deliverables or atomic concept notes here; link to projects/ and zettelkasten/ instead.

## How to operate

- **Do:** Create or edit area folders and their contents in line with each area’s README and AGENTS.md; link to projects, resources, and zettelkasten where relevant.
- **Don’t:** Create area content that is really project work or zettelkasten notes; don’t skip adding README.md and AGENTS.md when creating a new area.
